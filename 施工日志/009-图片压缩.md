# 施工日志 009 — 图片发送压缩

**日期**: 2026-05-27
**改动类型**: 功能增强

**改动内容**:
1. 新增 `compressImage()` 函数：Canvas 缩放 + JPEG 压缩
   - 最大尺寸 1920px（长边等比缩放）
   - JPEG quality 0.75
   - 返回压缩后的 dataUrl 和 Blob
2. `sendFile()` 改为 async：图片先调用 `compressImage()` 压缩，再分块发送
3. 压缩后文件名统一改为 `.jpg` 扩展名，mime 改为 `image/jpeg`
4. 压缩前后显示大小对比 toast（如 "图片已压缩: 5.2MB → 380KB"）
5. 压缩失败时回退到直接读取原文件

**原因**: 手机拍摄照片通常超过 5MB，base64 编码后约 7MB，通过 TURN 中继发送极慢。压缩后降至 200-500KB，传输时间大幅缩短。

**涉及文件**:
- index.html — 新增 compressImage()，sendFile() 改为 async + 压缩逻辑
