# 施工日志 008 — 连接稳定性修复

**日期**: 2026-05-27
**改动类型**: Bug 修复

**改动内容**:
1. Host 端连接逻辑修正：`peer.on('connection')` 内改为等 `c.on('open')` 再调用 `setupConnection()`，与 joiner 端保持一致，避免 ICE 未完成就显示"已连接"
2. 页面切后台处理：监听 `visibilitychange`，切后台时无活跃连接则主动销毁 Peer（防 iOS Safari 挂起 WebSocket 导致崩溃），回前台自动重建 Peer
3. 连接超时计时器：20s 更新状态提示"正在通过中继连接..."，60s 超时自动销毁 Peer 提示重试
4. `createRoom()` 明确设置 `isHost = true`，`joinRoom()` 明确设置 `isHost = false`
5. `window.beforeunload` 清理 Peer 和计时器

**原因**: 
- 问题1：iOS Safari 切后台到微信再切回时崩溃 — 因为 Safari 后台挂起 WebSocket，PeerJS 内部状态损坏
- 问题2：手机 B 卡在"正在连接"而 A 显示"连接成功" — Host 端在收到信令消息时就立即显示已连接，但 ICE 协商远未完成
- 问题3：两部手机 5G 连接需等待 40 秒以上 — 对称 NAT 必须走 TURN 中继，加拿大服务器延迟高

**涉及文件**:
- index.html — 修改 Host 连接逻辑、添加 visibility 处理、连接计时器、重连函数

- 无（当前版本未归档）
