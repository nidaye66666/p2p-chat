# p2p-burn-chat 设计方案

## 目标
轻量级 P2P 阅后即焚即时通讯应用。无需服务器，消息不落盘，双向删除。

## 技术选型
- **P2P 方案**: WebRTC + PeerJS (v1.5.2)
- **信令服务器**: PeerJS Cloud (0.peerjs.com)
- **STUN/TURN**: 4 STUN（国内 miwifi/qq + Google）+ 3 TURN（openrelay.metered.ca UDP/TCP:80/TCP:443），iceCandidatePoolSize=2
- **部署**: GitHub Pages (nidaye66666.github.io/p2p-chat)
- **存储**: 纯内存，不持久化

## 核心功能
- 6位邀请码连接
- 文字消息（阅后即焚，3-60秒可配置）
- 语音消息（按住录音，MediaRecorder API）
- 图片/文件传输（分块 base64，DataChannel）
- 双向焚毁同步
- 连接断开检测

## UI 设计
- GitHub 风格暗色主题 (#0d1117 背景, #58a6ff 强调色)
- 移动端适配 (100dvh, flex 布局)
- Toast 通知替代 alert
- 语音按键: 长按 200ms 触发录音

## 安全
- WebRTC DTLS 加密（传输层）
- 6位邀请码访问控制
- 消息不落盘
- 后续待加: 邀请码暴力破解防护、端到端 AES 加密
